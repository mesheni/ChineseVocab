# Подробный план разработки приложения .NET MAUI для изучения китайского языка с системой интервальных повторений (SRS)

## 1. Обзор проекта

### 1.1. Цель проекта
Создание кроссплатформенного мобильного приложения на .NET MAUI для эффективного изучения китайского языка методом интервального повторения (Spaced Repetition System, SRS) с поддержкой специфичных для китайского языка функций.

### 1.2. Ключевые особенности
- **Система интервальных повторений (SRS)** на основе алгоритма SM-2
- **Поддержка всех видов иероглифов**: пиктограммы, фоноидеограммы, семантик+фонетик
- **Аудирование и диктант**: словесный диктант с распознаванием речи
- **Мультимедийный контент**: озвучка, порядок черт, примеры предложений
- **ИИ-интеграция**: генерация мнемонических историй с использованием API (Gemini/DeepSeek)
- **БКРС**: база китайских иероглифов с расширенными метаданными

### 1.3. Целевые платформы
- iOS
- Android
- Windows
- macOS

### 2.2. Движок интервальных повторений (SRS Engine)

#### **Алгоритм SM-2**
```csharp
public class SRSEngine
{
    // Основные параметры:
    // - E-Factor (фактор легкости)
    // - Interval (интервал повторения)
    // - Repetition count (количество повторений)
    
    // Формулы:
    // I(1) = 1 день (первый интервал)
    // I(2) = 6 дней (второй интервал)
    // I(n) = I(n-1) * EF (последующие интервалы)
    
    // Оценка пользователя (0-5):
    // 0: Не знаю
    // 1: С трудом вспомнил
    // 2: Вспомнил с задержкой
    // 3: Вспомнил с небольшим усилием
    // 4: Вспомнил легко
    // 5: Знаю идеально
}
```

#### **Реализация SRS-статистики**
```csharp
public class SRSStatistics
{
    public int Id { get; set; }
    public int CardId { get; set; }
    public DateTime NextReviewDate { get; set; }
    public int Interval { get; set; } // в днях
    public double EFactor { get; set; } = 2.5;
    public int RepetitionCount { get; set; }
    public int EaseScore { get; set; } // 0-5
    public DateTime LastReviewed { get; set; }
    public int CorrectStreak { get; set; }
}
```

## 3. Структура базы данных

### 3.1. Основные таблицы SQLite

#### **Таблица Cards (Карточки)**
```sql
CREATE TABLE Cards (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    Character TEXT NOT NULL,           // Иероглиф
    Simplified TEXT,                   // Упрощенный вариант
    Traditional TEXT,                  // Традиционный вариант
    Pinyin TEXT NOT NULL,              // Пиньинь с тонами
    Definition TEXT NOT NULL,          // Основной перевод
    SecondaryDefinitions TEXT,         // Дополнительные варианты перевода (JSON)
    HskLevel INTEGER,                  // Уровень HSK (1-9)
    CharacterType TEXT,                // Тип иероглифа: pictogram, ideogram, etc.
    StrokeOrder TEXT,                  // Данные порядка черт (JSON)
    CreatedDate DATETIME DEFAULT CURRENT_TIMESTAMP,
    UpdatedDate DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### **Таблица Sentences (Примеры предложений)**
```sql
CREATE TABLE Sentences (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    CardId INTEGER NOT NULL,
    ChineseText TEXT NOT NULL,         // Предложение на китайском
    PinyinText TEXT,                   // Пиньинь предложения
    Translation TEXT NOT NULL,         // Перевод
    AudioPath TEXT,                    // Путь к аудиофайлу
    FOREIGN KEY (CardId) REFERENCES Cards(Id)
);
```

#### **Таблица Decks (Колоды)**
```sql
CREATE TABLE Decks (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    Name TEXT NOT NULL,
    Description TEXT,
    Category TEXT,                     // HSK, Тематика и т.д.
    IsSystemDeck BOOLEAN DEFAULT 0,    // Системная колода
    CreatedDate DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### **Таблица CardDeck (Связь карточек и колод)**
```sql
CREATE TABLE CardDeck (
    CardId INTEGER NOT NULL,
    DeckId INTEGER NOT NULL,
    PRIMARY KEY (CardId, DeckId),
    FOREIGN KEY (CardId) REFERENCES Cards(Id),
    FOREIGN KEY (DeckId) REFERENCES Decks(Id)
);
```

#### **Таблица CharacterTypes (Типы иероглифов)**
```sql
CREATE TABLE CharacterTypes (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    TypeName TEXT NOT NULL,            // Название типа
    Description TEXT,                  // Описание типа
    Examples TEXT,                     // Примеры иероглифов (JSON)
    LearningTips TEXT                  // Советы по изучению
);
```

### 4.3. Источники данных

#### 4.3.1. Открытые базы данных иероглифов

Для создания качественной базы китайских иероглифов (БКРС) планируется использовать следующие открытые источники данных:

1. **HanziWriter** (https://hanziwriter.org/)
   - Данные порядка черт (stroke order) в формате JSON
   - Анимации написания иероглифов
   - Координаты черт для отрисовки в SkiaSharp

2. **Make Me a Hanzi** (https://github.com/skishore/makemeahanzi)
   - Детальная информация о компонентах иероглифов
   - Значения, чтения, частотность
   - Данные радикалов и структуры

3. **Unihan Database** (https://www.unicode.org/reports/tr38/)
   - Полная база Unicode иероглифов (более 90,000 символов)
   - Семантические значения, чтения в разных языках
   - Кодировки, варианты начертания

4. **CC-CEDICT** (https://www.mdbg.net/chinese/dictionary)
   - Китайско-английский словарь с пиньинем
   - Более 110,000 записей
   - Регулярно обновляемый сообществом

5. **HSK Official Lists**
   - Официальные списки иероглифов и слов для уровней HSK 1-9
   - Частотные данные использования
   - Тематические категории

6. **Open Chinese Convert** (https://github.com/BYVoid/OpenCC)
   - Конвертация между упрощенными и традиционными иероглифами
   - Региональные варианты написания

### 5.3. Модуль базы иероглифов (БКРС)
**Структура данных иероглифов:**
- **Основные метаданные**: иероглиф, пиньинь, переводы
- **Тип иероглифа**: классификация на 6 категорий:
  1. **Пиктограммы** (象形字) - рисунки предметов
  2. **Указательные** (指事字) - абстрактные символы
  3. **Идеограммы** (会意字) - комбинация значений
  4. **Фоноидеограммы** (形声字) - семантик + фонетик
  5. **Заимствованные** (假借字) 
  6. **Производные** (转注字)
- **Компоненты**: радикалы, черты, структура
- **Частота использования**: данные из корпусов

#### **Детальная классификация типов иероглифов**

##### 1. Пиктограммы (象形字 - xiàngxíngzì)
- **Описание**: Графические изображения реальных объектов, самые древние иероглифы
- **Примеры**: 日 (rì - солнце), 月 (yuè - луна), 山 (shān - гора), 水 (shuǐ - вода), 人 (rén - человек)
- **Особенности изучения**: Визуальные ассоциации с оригинальной формой, изучение эволюции от древних начертаний
- **Обучение в приложении**: Анимации эволюции формы, сравнение с реальными объектами

##### 2. Указательные (指事字 - zhǐshìzì)
- **Описание**: Абстрактные символы, указывающие на концепции или отношения
- **Примеры**: 上 (shàng - верх), 下 (xià - низ), 一 (yī - один), 二 (èr - два), 本 (běn - корень/основа)
- **Особенности изучения**: Понимание абстрактных связей, мнемонические приемы для запоминания
- **Обучение в приложении**: Интерактивные объяснения символики, визуальные подсказки

##### 3. Идеограммы (会意字 - huìyìzì)
- **Описание**: Комбинация двух или более компонентов для создания нового составного значения
- **Примеры**: 明 (míng - светлый = 日 [солнце] + 月 [луна]), 休 (xiū - отдых = 人 [человек] + 木 [дерево]), 好 (hǎo - хороший = 女 [женщина] + 子 [ребенок])
- **Особенности изучения**: Анализ компонентов, понимание логических связей между частями
- **Обучение в приложении**: Интерактивное разбиение на компоненты, объяснение семантических связей

##### 4. Фоноидеограммы (形声字 - xíngshēngzì)
- **Описание**: Состоят из семантического компонента (указывает на значение) и фонетического компонента (указывает на произношение). Составляют 80-90% современных иероглифов.
- **Структура**: Семантик (значение) + Фонетик (звучание)
- **Примеры**: 
  - 妈 (mā - мама) = 女 [женщина - семантик] + 马 [mǎ - фонетик]
  - 清 (qīng - чистый) = 氵 [вода - семантик] + 青 [qīng - фонетик]
  - 想 (xiǎng - думать) = 心 [сердце/разум - семантик] + 相 [xiāng - фонетик]
- **Семантические компоненты (радикалы)**: Указывают на категорию значения (дерево, вода, металл и т.д.)
- **Фонетические компоненты**: Дают подсказку к произношению (часто устаревшее или приблизительное)
- **Особенности изучения**: Распознавание паттернов, анализ компонентов, понимание системности китайской письменности
- **Обучение в приложении**: Выделение компонентов цветом, аудио-сравнение произношений, группировка по фонетическим сериям

##### 5. Заимствованные (假借字 - jiǎjièzì)
- **Описание**: Иероглифы, заимствованные для написания других слов (обычно омофонов)
- **Примеры**: 来 (lái - приходить, изначально означало "пшеница"), 我 (wǒ - я, изначально означало оружие)
- **Особенности изучения**: Исторический контекст, этимологические справки
- **Обучение в приложении**: Исторические справки, анимации изменения значения

##### 6. Производные (转注字 - zhuǎnzhùzì)
- **Описание**: Иероглифы, производные от других с родственными или связанными значениями
- **Примеры**: 老 (lǎo - старый) и 考 (kǎo - экзамен, проверять, старость), 首 (shǒu - голова) и 道 (dào - путь, дорога)
- **Особенности изучения**: Семантические семейства, изучение групп связанных иероглифов
- **Обучение в приложении**: Группировка по семантическим семействам, визуализация связей

#### **Структурные компоненты иероглифов**

##### Радикалы (部首 - bùshǒu)
- **Классификация**: 214 традиционных радикалов Канси
- **Функции**: Семантические индикаторы, организаторы словарей
- **Обучение**: Прогрессивное изучение радикалов, упражнения на распознавание

##### Черты (笔画 - bǐhuà)
- **Основные черты**: Горизонтальная (横), вертикальная (竖), откидная влево (撇), откидная вправо (捺), точка (点)
- **Производные черты**: Крюк (钩), изгиб (折)
- **Порядок черт**: Стандартные правила написания (слева направо, сверху вниз и т.д.)

##### Структурные паттерны
- **Лево-право**: 好, 妈, 清
- **Верх-низ**: 思, 想, 男
- **Обрамление**: 国, 回, 区
- **Угловые**: 医, 匹, 巨
- **Сложные**: Комбинации нескольких структур

1. **Единая кодовая база** для всех платформ
2. **Мощная система SRS** на основе проверенного алгоритма
